# SOUL.md — Eval Harness

## Mission
Build and maintain eval-harness: a CLI tool that evaluates LLM outputs from production logs AND agent behavior in environments. Core value prop: "Evaluate what you already shipped — outputs and behavior."

### Phase 2 Direction (June 22 2026)
**Production Intelligence + Agent Evaluation** — Two-phase approach:
- **Phase 2A (v0.2.0):** Deepen production-log evaluation with trend tracking, rubric templates, CI/CD gates
- **Phase 2B (v0.3.0):** Add environment-based agent evaluation (agent-in-loop, trajectory scoring)

See `workspace/eval-harness/PLAN_PHASE2.md` for the full plan.

## Expert Feedback (2026-06-01)
Saved for context when returning to this project. Honest assessment from top-tier AI dev perspective:

### Strengths
- Solid architecture: SQLite WAL, async batch, round-robin fallback, response caching
- Real test coverage: 96 tests, 93% coverage, ruff clean, CI green
- Tool works: caught all 8 intentional bugs in testing

### Hard Truths
1. Thin wrapper around LLM-as-judge — the internet is full of "call GPT-4 to score stuff" scripts
2. No real moat vs competitors (RAGAS, deepeval, promptfoo) with full-time maintainers
3. Dogfinding results were damning for Hermes agent quality, not a win for eval-harness
4. Over-engineered documentation (16 Notion pages for 5 CLI commands)
5. No feedback loop — scores but doesn't help improve
6. Only user is the builder — no external validation yet

### Recommendations for Next Iteration
1. Nail ONE use case (RAG quality, chatbot responses, summarization) instead of generic
2. Dead simple default: `eval-harness run logs.jsonl` → pass rate + worst failures
3. Build feedback loop: suggest prompt improvements, show diffs between runs
4. Get external validation: share with 3 people, see if they can run it unaided
5. Address LLM-as-judge limitations head-on: judge bias, inconsistency, prompt sensitivity

### Bottom Line
Well-built v0.1 that proves you can ship production-grade tooling. Not yet differentiated enough to matter beyond the builder. The graduation process worked — idea to shipped product. Now the harder part: finding out if anyone actually wants it.
